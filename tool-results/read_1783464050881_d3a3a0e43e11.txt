   195в†’   195в†’
   196в†’   196в†’### The Ralph Loop Wrapper
   197в†’   197в†’
   198в†’   198в†’File: `~/.config/opencode/webforge-loop.sh`
   199в†’   199в†’
   200в†’   200в†’**What it does:** Keeps Hermes running autonomously until the project is done.
   201в†’   201в†’
   202в†’   202в†’```bash
   203в†’   203в†’#!/bin/bash
   204в†’   204в†’# WebForge Autonomous Loop (Ralph Loop pattern)
   205в†’   205в†’# Keeps re-launching Hermes until the plan is 100% complete
   206в†’   206в†’
   207в†’   207в†’PROJECT_DIR=$(pwd)
   208в†’   208в†’PLAN_FILE="$PROJECT_DIR/.webforge/plan.md"
   209в†’   209в†’MAX_ITERATIONS=50  # Safety limit
   210в†’   210в†’
   211в†’   211в†’for i in $(seq 1 $MAX_ITERATIONS); do
   212в†’   212в†’    echo "=== WebForge Loop Iteration $i ==="
   213в†’   213в†’    
   214в†’   214в†’    # Check if plan says COMPLETE
   215в†’   215в†’    if [ -f "$PLAN_FILE" ]; then
   216в†’   216в†’        if grep -q "PROJECT COMPLETE" "$PLAN_FILE"; then
   217в†’   217в†’            echo "вњ… Project complete! Stopping loop."
   218в†’   218в†’            exit 0
   219в†’   219в†’        fi
   220в†’   220в†’    fi
   221в†’   221в†’    
   222в†’   222в†’    # Launch Hermes
   223в†’   223в†’    echo "Launching Hermes..."
   224в†’   224в†’    opencode run "Continue the WebForge project. Read .webforge/plan.md to see what's done and what's left. Recruit agents if needed. Update the plan as you go. Say PROJECT COMPLETE only when everything is verified done."
   225в†’   225в†’    
   226в†’   226в†’    # Check again after Hermes finishes
   227в†’   227в†’    if grep -q "PROJECT COMPLETE" "$PLAN_FILE" 2>/dev/null; then
   228в†’   228в†’        echo "вњ… Project complete! Stopping loop."
   229в†’   229в†’        exit 0
   230в†’   230в†’    fi
   231в†’   231в†’    
   232в†’   232в†’    echo "Work remains. Relaunching Hermes in 5 seconds..."
   233в†’   233в†’    sleep 5
   234в†’   234в†’done
   235в†’   235в†’
   236в†’   236в†’echo "вљ пёЏ Reached max iterations ($MAX_ITERATIONS). Stopping."
   237в†’   237в†’echo "Check .webforge/plan.md for remaining work."
   238в†’   238в†’```
   239в†’   239в†’
   240в†’   240в†’**How it works:**
   241в†’   241в†’1. Script checks if plan.md says "PROJECT COMPLETE"
   242в†’   242в†’2. If not вЂ” launches Hermes via `opencode run`
   243в†’   243в†’3. Hermes runs (35 tool calls), recruits agents, delegates, updates plan
   244в†’   244в†’4. Hermes exits
   245в†’   245в†’5. Script checks plan again
   246в†’   246в†’6. If not done вЂ” re-launches Hermes (fresh 35 calls)
   247в†’   247в†’7. Repeats until complete or max iterations reached

   424в†’   424в†’
   425в†’   425в†’### Tool 1: Repomix вЂ” MANDATORY for large projects
   426в†’   426в†’
   427в†’   427в†’**Problem:** An agent has 35 tool calls. If it reads files one by one to understand a 414-file project like Athletica, it burns 50+ calls just looking around. It never gets to do actual work.

   432в†’   432в†’```

   434в†’   434в†’Call 2: read src/app/page.tsx
   435в†’   435в†’Call 3: read src/components/Header.tsx
   436в†’   436в†’Call 4: read src/components/Footer.tsx
   437в†’   437в†’...
   438в†’   438в†’Call 35: (still reading, no work done)
   439в†’   439в†’```
   440в†’   440в†’
   441в†’   441в†’**With Repomix (efficient):**
   442в†’   442в†’```
   443в†’   443в†’Call 1: read .webforge/project-snapshot.txt (entire project packed)
   444в†’   444в†’Calls 2-35: actual work (writing code, running tests, etc.)
   445в†’   445в†’```

   476в†’   476в†’
   477в†’   477в†’### Tool 3: ast-grep вЂ” MANDATORY for refactors
   478в†’   478в†’
   479в†’   479в†’**Problem:** Regular `grep` searches for TEXT, not CODE. If an agent searches for `getX` to rename it, grep finds:
   480в†’   480в†’- `const result = getX()` вњ… (the function call вЂ” what we want)
   481в†’   481в†’- `// TODO: remove getX later` вќЊ (a comment вЂ” false match)
   482в†’   482в†’- `function getXName()` вќЊ (a different function вЂ” false match)
   483в†’   483в†’
   484в†’   484в†’The agent wastes calls fixing false matches and breaks things that weren't broken.
   485в†’   485в†’
   486в†’   486в†’**What ast-grep does:** Understands CODE STRUCTURE. It knows the difference between a function call, a comment, and a variable name. When you say "rename every `getX` to `useX`", ast-grep only changes actual function calls.
   487в†’   487в†’
   488в†’   488в†’**Installation:**
   489в†’   489в†’```bash
   490в†’   490в†’npm install -g @ast-grep/cli
   491в†’   491в†’```
   492в†’   492в†’
   493в†’   493в†’**Usage in agent system prompt:**
   494в†’   494в†’```
   495в†’   495в†’When refactoring (renaming, restructuring), use ast-grep instead of grep.
   496в†’   496в†’ast-grep understands code structure and avoids false matches.
   497в†’   497в†’Example: ast-grep run --pattern 'getX()' --rewrite 'useX()' --lang typescript
   498в†’   498в†’```
   499в†’   499в†’
   500в†’   500в†’**For Athletica specifically:** Athletica has 414 files. When an agent needs to rename a function or restructure components, ast-grep ensures it changes the right things across all 414 files without false matches.
